# 🚀 RAG系统快速开始指南

## 立即开始（3步）

### 1. 确认环境已修复 ✅

环境已经修复完成，所有依赖包版本正确。

### 2. 选择数据源

```bash
python switch_data_source.py
```

选择你要使用的数据类型:
- **选项1**: 小说模式（长文本）
- **选项2**: 短对话模式（聊天记录）
- **选项3**: 自定义JSONL文件

### 3. 启动系统

```bash
python main.py
```

系统会自动:
- 加载你选择的数据文件
- 构建或加载向量库
- 初始化BM25检索器
- 加载Reranker模型

然后就可以开始提问了！

## 如果你还没有数据

### 准备小说数据

1. 准备一个TXT小说文件
2. 编辑 `qiefen_final.py`:
   ```python
   INPUT_TXT = r"你的小说路径.txt"
   OUTPUT_JSONL = r"输出路径.jsonl"
   ```
3. 运行: `python qiefen_final.py`
4. 等待处理完成（可能需要较长时间）
5. 使用 `switch_data_source.py` 选择生成的JSONL文件

### 准备短对话数据

1. 准备一个TXT聊天记录文件，格式:
   ```
   说话人1: 内容1
   说话人2: 内容2
   说话人1: 内容3
   ```

2. 运行: `python annotate_short_text.py`

3. 按提示输入文件路径

4. 等待处理完成

5. 使用 `switch_data_source.py` 选择生成的JSONL文件

## 系统功能

### 已启用的优化

- ✅ **Reranker重排序**: 提升检索精度
- ✅ **BM25混合检索**: 结合语义和关键词匹配
- ✅ **意图感知优化**: 根据问题类型优化检索
- ✅ **语义关系验证**: 深度理解问题语义

### 可配置参数

编辑 `config.py` 调整:
- `TOP_K`: 返回结果数量（默认10）
- `DISTANCE_THRESHOLD`: 相似度阈值（默认0.6）
- `ENABLE_RERANKER`: 是否启用Reranker（默认True）
- `ENABLE_BM25`: 是否启用BM25（默认True）
- `ENABLE_INTENT_AWARE`: 是否启用意图感知（默认True）

## 评估模式

系统默认开启评估模式，会显示:
- 检索到的chunk数量
- Reranker重排序结果
- 意图分类结果
- 语义关系提取结果

在系统运行时可以切换:
```
eval on   # 开启评估模式
eval off  # 关闭评估模式
config    # 查看当前配置
quit      # 退出系统
```

## 故障排除

### 问题: 提示文件不存在

**解决**:
```bash
python switch_data_source.py
# 选择选项 4 查看当前配置
# 选择选项 3 指定正确的文件路径
```

### 问题: 查询结果不准确

**解决**:
1. 确认使用了正确的数据源（小说 vs 短对话）
2. 删除向量库文件夹重建:
   ```bash
   # 删除 chroma_rag_db 或 chroma_chat_db 文件夹
   # 重新运行 python main.py
   ```

### 问题: 系统运行缓慢

**原因**: 首次运行需要:
- 下载embedding模型（约200MB）
- 构建向量库（取决于数据量）
- 加载Reranker模型（约400MB）

**解决**: 后续运行会快很多，因为模型和向量库都已缓存
