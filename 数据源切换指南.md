# 📚 RAG系统数据源切换指南

## 概述

RAG系统现在支持两种数据类型:
- **小说模式**: 处理长文本小说内容
- **短对话模式**: 处理聊天记录、对话内容

## 快速开始

### 方法1: 使用切换工具（推荐）

```bash
python switch_data_source.py
```

按照提示选择:
1. 切换到小说模式
2. 切换到短对话模式  
3. 指定自定义JSONL文件
4. 查看当前配置

### 方法2: 手动配置

编辑 `text_type_config.py` 中的默认文本类型:

```python
# 在 get_config() 函数中修改默认值
_global_config = TextTypeConfig(TEXT_TYPE_CHAT)  # 或 TEXT_TYPE_NOVEL
```

## 完整工作流程

### 小说数据处理流程

1. **准备原始TXT文件**
   ```
   例如: C:\Users\xxx\Downloads\逆天邪神.txt
   ```

2. **运行切分和标注脚本**
   ```bash
   # 编辑 qiefen_final.py 中的路径
   INPUT_TXT = r"C:\Users\xxx\Downloads\逆天邪神.txt"
   OUTPUT_JSONL = r"D:\rag\venv\逆天邪神_自动标注版_最终.jsonl"
   
   # 运行脚本
   python qiefen_final.py
   ```

3. **切换到小说模式**
   ```bash
   python switch_data_source.py
   # 选择选项 1 或 3（指定自定义文件）
   ```

4. **运行RAG系统**
   ```bash
   python main.py
   ```

### 短对话数据处理流程

1. **准备原始TXT文件（带说话人格式）**
   ```
   格式示例:
   元: 明年你高考要是能考到西农我给你发杯奶茶钱[旺柴]
   菻荺: 我一定要考到西农吗？考其他的不可以[旺柴]
   元: 不行
   菻荺: [Emm]
   ```

2. **运行短对话标注脚本**
   ```bash
   python annotate_short_text.py
   ```
   
   按照提示:
   - 输入TXT文件路径
   - 输入输出JSONL文件路径
   - 等待处理完成

3. **切换到短对话模式**
   ```bash
   python switch_data_source.py
   # 选择选项 2 或 3（指定自定义文件）
   ```

4. **运行RAG系统**
   ```bash
   python main.py
   ```

## 重要提示

### 向量库管理

- 每次切换数据源后，系统会自动使用对应的向量库
- 如果向量库不存在，系统会自动创建
- 如果数据文件更新，需要删除旧的向量库文件夹让系统重建

### 数据格式要求

**小说JSONL格式**:
```json
{
  "chunk_id": 1,
  "characters": ["云澈", "夏倾月"],
  "types": ["事件", "心理"],
  "keywords": ["修炼", "突破", "玄力"],
  "summary": "云澈突破玄力境界",
  "importance": "high",
  "content": "..."
}
```

**短对话JSONL格式**:
```json
{
  "chunk_id": 1,
  "characters": ["元", "菻荺"],
  "types": ["情感表达", "问题讨论"],
  "keywords": ["高考", "西农", "奶茶"],
  "summary": "讨论高考目标学校",
  "importance": "middle",
  "content": "元: 明年你高考要是能考到西农我给你发杯奶茶钱\n菻荺: 我一定要考到西农吗？考其他的不可以\n元: 不行\n菻荺: [Emm]",
  "topic_id": 1,
  "speakers": ["元", "菻荺"]
}
```

## 常见问题

### Q: 切换数据源后需要重启系统吗？
A: 是的，需要重新运行 `python main.py`

### Q: 可以同时使用多个数据源吗？
A: 不可以，每次只能使用一个数据源。如需切换，使用 `switch_data_source.py`

### Q: 向量库占用空间太大怎么办？
A: 可以删除不用的向量库文件夹（如 `chroma_rag_db` 或 `chroma_chat_db`）

### Q: 如何验证当前使用的数据源？
A: 运行 `python switch_data_source.py` 选择选项4查看当前配置

## 故障排除

### 问题: 运行main.py时提示文件不存在
**解决方案**: 
1. 运行 `python switch_data_source.py` 查看当前配置
2. 确认数据文件路径正确
3. 使用选项3指定正确的文件路径

### 问题: 查询结果不符合预期
**解决方案**:
1. 确认使用了正确的数据源（小说 vs 短对话）
2. 检查向量库是否是最新的（删除旧向量库重建）
3. 查看数据文件内容是否正确标注

### 问题: KeyError或其他运行时错误
**解决方案**:
1. 确认已修复 `semantic_relation_extractor.py` 和 `prompts.py`
2. 重新运行 `python main.py`
3. 如果问题持续，检查JSONL文件格式是否正确

## 性能优化建议

1. **小说模式**: 适合长文本，chunk大小250字左右
2. **短对话模式**: 适合短对话，使用话题分段，每段3-8轮对话
3. **向量库**: 首次构建较慢，后续查询很快
4. **Reranker**: 建议启用，显著提升检索质量
