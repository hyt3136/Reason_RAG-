## 旧方法 (short_text_processor.py)

### 策略
1. 把每一行当作独立的message
2. 计算相邻行的向量相似度
3. 在相似度低于阈值的地方切分
4. 结果：产生大量超短段落

### 问题
- ❌ 每行都是独立的，导致过度切分
- ❌ 短文本的向量相似度不稳定
- ❌ 产生的段落太短，没有上下文
- ❌ LLM无法有效处理这么短的文本

### 代码逻辑
```python
# 旧方法：逐行比较
for i in range(1, len(messages)):
    similarity = calculate_similarity(embeddings[i-1], embeddings[i])
    
    if similarity < THRESHOLD:  # 每两行之间都可能切分
        create_new_segment()
```

**结果**: 1134个段落，每个2-13字符

## 新方法 (short_text_processor_v2.py)

### 策略（你建议的方法）
1. **先合并为长对话**（3-10轮为一个段落）
2. **然后语义切分**（只处理过长的段落）
3. **添加滑动窗口上下文**
4. **按照长对话逻辑处理**

### 优势
- ✅ 每个段落有足够的上下文（100-300字符）
- ✅ 保证最少3轮对话，最多10轮
- ✅ 使用滑动窗口保证连续性
- ✅ LLM可以有效理解和标注

### 代码逻辑
```python
# 新方法：先合并，再切分
while i < len(turns):
    current_turns = []
    
    # 至少收集3轮
    while len(current_turns) < MIN_TURNS:
        current_turns.append(turns[i])
        i += 1
    
    # 继续收集直到达到10轮或300字符
    while len(current_turns) < MAX_TURNS and current_chars < MAX_DIALOG_LENGTH:
        current_turns.append(turns[i])
        i += 1
    
    # 创建对话段
    create_dialog_segment(current_turns)
    
    # 回退2轮，实现滑动窗口
    i -= OVERLAP_TURNS
```

**结果**: 约50-100个段落，每个100-300字符，3-10轮对话

## 参数配置

### 新方法参数
```python
# 合并配置
MIN_DIALOG_LENGTH = 100  # 最小对话长度（字符）
MAX_DIALOG_LENGTH = 300  # 最大对话长度（字符）
MIN_TURNS = 3  # 最少对话轮数
MAX_TURNS = 10  # 最多对话轮数

# 滑动窗口配置
OVERLAP_TURNS = 2  # 重叠轮数（前后各保留N轮）
```

### 为什么这样设置？
- **MIN_TURNS = 3**: 至少3轮对话才能理解上下文
- **MAX_TURNS = 10**: 太多轮会导致话题混杂
- **OVERLAP_TURNS = 2**: 保证相邻段落有连续性
- **MIN_DIALOG_LENGTH = 100**: 保证有足够的信息量
- **MAX_DIALOG_LENGTH = 300**: 避免段落过长
