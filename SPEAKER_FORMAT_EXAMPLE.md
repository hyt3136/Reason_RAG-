# 说话人格式使用示例

## 📝 你的对话格式

你提供的格式：
```
元: 明年你高考要是能考到西农我给你发杯奶茶钱[旺柴]
菻荺: 我一定要考到西农吗？考其他的不可以[旺柴]
元: 不行
菻荺: [Emm]
```

## ✅ 系统支持特性

### 1. 自动识别说话人
- ✅ 支持英文冒号 `:` 
- ✅ 支持中文冒号 `：`
- ✅ 自动提取说话人名称（如 "元"、"菻荺"）

### 2. 自动清理表情符号
- ✅ 自动移除 `[旺柴]`、`[Emm]`、`[微笑]` 等方括号表情
- ✅ 保留纯文本内容用于语义分析
- ✅ 可通过配置关闭此功能

### 3. 保留说话人信息
- ✅ 在增强版JSONL中保存说话人列表
- ✅ 用于后续的角色分析和关系提取
- ✅ 支持多人对话场景

## ⚙️ 配置选项

在 `config.py` 中可以调整：

```python
# 说话人格式支持
ENABLE_SPEAKER_DETECTION = True  # 是否自动检测说话人
CLEAN_EMOJIS = True  # 是否清理表情符号
SPEAKER_SEPARATOR = [":", "："]  # 说话人分隔符
```

### 如果不想清理表情符号

修改 `config.py`：
```python
CLEAN_EMOJIS = False  # 保留表情符号
```

这样输出会保留 `[旺柴]`、`[Emm]` 等表情。

## 💡 使用建议

### 1. 保持格式一致
- 统一使用英文冒号或中文冒号
- 每行一条对话
- 说话人名称保持一致

### 2. 说话人命名
- ✅ 推荐：使用简短的昵称（如 "元"、"菻荺"）
- ✅ 可以：使用中文名（如 "小明"、"小红"）
- ⚠️ 避免：过长的名称或包含特殊字符

### 3. 表情符号处理
- 如果表情符号有语义价值，可以关闭清理功能
- 如果只是装饰性表情，建议保持清理开启

### 4. 对话质量
- 确保每条对话有实际内容
- 避免纯表情的行（如只有 `[Emm]`）
- 保持对话的连贯性

## 🔧 常见问题

### Q1: 系统没有识别到说话人？
**A**: 检查以下几点：
1. 是否使用了冒号分隔符（`:` 或 `：`）
2. 冒号前是否有说话人名称
3. 是否超过50%的行包含冒号

### Q2: 表情符号没有被清理？
**A**: 检查 `config.py` 中的 `CLEAN_EMOJIS` 是否为 `True`

### Q3: 说话人名称显示不正确？
**A**: 确保冒号前的说话人名称：
- 不包含空格
- 不包含特殊字符
- 保持一致性

### Q4: 如何处理群聊（多人对话）？
**A**: 系统完全支持多人对话：
```
小明: 今天去哪玩
小红: 去公园吧
小李: 我也想去
小明: 那就一起去
```
